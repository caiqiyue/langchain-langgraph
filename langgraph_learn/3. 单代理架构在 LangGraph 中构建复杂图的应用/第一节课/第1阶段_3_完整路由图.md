## <center>第1阶段、Router Agent与条件路由</center>

### 1. 结合结构化输出构建路由图

&nbsp;&nbsp;&nbsp;&nbsp;三种不同的结构化输出方法，我们更常使用的是用`Pydantic`来处理路由决策。在这种策略下，我们可以通过定义一个包含`Union`类型属性的父模型来灵活地从多种模式中选择适当的路由分支。

&nbsp;&nbsp;&nbsp;&nbsp;例如，如果我们想根据输出决定是查询数据库还是直接回答问题，可以创建一个统一的模型来封装可能的输出类型：

```python
class FinalResponse(BaseModel):
    final_output: Union[UserInfo, ConversationalResponse]
```

### 2. 完整的 Router Agent 案例

&nbsp;&nbsp;&nbsp;&nbsp;整个流程是：用户输入 -> chat_with_model 节点（结构化输出）-> Router Function（判断类型）-> 根据类型路由到不同节点（insert_db 或 final_answer）-> 结束。

&nbsp;&nbsp;&nbsp;&nbsp;通过这种设计，系统可以可靠地解释大模型的决定并采取行动。Router Agent 的优势就是可以精准的控制程序链路中的每一个细节。

### 3. 拓展思考

&nbsp;&nbsp;&nbsp;&nbsp;由于 Router Agent 是预设的分支结构，需要根据用户的输入请求灵活适配不同的场景。结构化输出确保了系统可以可靠地解释大模型的决定并采取行动。当需要在条件分支中执行更复杂的工具调用时，就需要引入 Tool Calling Agent。
