## <center>第3阶段、事件流的应用</center>

### 1. LangGraph 流模式概述

&emsp;&emsp;LangGraph 提供了多种流模式来处理 Agent 的输出，主要包括 `stream`（同步）和 `astream`（异步）两种调用方式。流模式允许实时获取 Agent 执行过程中的中间结果，而非等待完整执行完成后才返回。

### 2. 流模式类型

&emsp;&emsp;LangGraph 支持四种主要流模式：

&emsp;&emsp;**values 模式**：返回图中每次迭代后的完整状态值，适合需要查看完整中间状态的场景。

&emsp;&emsp;**updates 模式**：返回每次迭代后的状态更新部分，格式为 `{node_name: values}`，适合只需要关注特定节点更新的场景。

&emsp;&emsp;**debug 模式**：返回详细的调试信息，包括执行时间、节点名称等，适合调试和性能分析。

&emsp;&emsp;**messages 模式**：以消息流的形式返回输出，专为聊天模型设计，可以实时获取生成的 token。

### 3. LangGraph 事件机制

&emsp;&emsp;使用 `astream_events` 方法可以追踪 Agent 执行过程中的所有事件，包括链开始/结束、聊天模型开始/生成/结束、工具开始/结束/执行中等事件类型。

&emsp;&emsp;每个事件包含 `event`（事件类型）、`name`（事件名称）、`data`（事件数据）、`metadata`（元数据）等字段。

### 4. 事件过滤与处理

&emsp;&emsp;可以根据事件类型、名称、标签等条件过滤事件，实现只获取感兴趣的事件数据。常见的过滤场景包括：只获取聊天模型的 token 流、获取特定节点的事件、获取工具执行结果等。
