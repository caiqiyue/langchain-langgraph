Reducer 函数机制

🔧Reducer 原理

当前状态
+
新操作
→
Reducer 函数
→
新状态
  • 根据输入生成新的状态
  • 定义状态的更新逻辑
  • 控制状态的合并方式

🏷️Annotated 与 operator.add

# 使用 Annotated 定义 Reducer
from typing import Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]
    count: int

默认行为

相同 key 被新值覆盖

operator.add

列表追加模式，新值添加到现有列表

📝代码示例

# 默认覆盖更新
state = {"messages": ["a"]}
new_state = {"messages": ["b"]}
# 结果: {"messages": ["b"]}

# 使用 operator.add
from typing import Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]

# reducer: lambda old, new: old + new
# 结果: {"messages": ["a", "b"]}

⚡核心优势

  • 灵活的状态更新策略
  • 支持列表、计数器等多种类型
  • 可自定义合并逻辑
  • 适合消息历史等场景

Reducer = 状态更新策略，通过 Annotated 指定自定义合并函数
