🔧 中间件四大分类

📊 Monitor 监控类
观察执行状态、日志记录
解决问题：调试困难、缺乏可观测性。性能影响极小，几乎无感知。

常见中间件：

  • • LangChainTracingMiddleware
  • • ToolCallLogger
  • • CostTrackingMiddleware
性能影响 < 1ms
✏️ Modify 修改类
修改输入/输出、上下文管理
解决问题：上下文窗口溢出、Prompt优化。性能影响取决于处理逻辑。

常见中间件：

  • • SummarizationMiddleware
  • • DynamicPromptMiddleware
  • • ContextEditingMiddleware
性能影响 1-50ms
🎛️ Control 控制类
流程阻断、人工介入
解决问题：AI幻觉、高风险操作失控。可能包含等待时间。

常见中间件：

  • • HumanInTheLoopMiddleware
  • • RetryMiddleware
  • • CircuitBreakerMiddleware
性能影响 5-20ms
🛡️ Enforce 强制类
安全过滤、限流、合规检查
解决问题：数据泄露、API滥用。性能影响较小，必须优先执行。

常见中间件：

  • • PIIMiddleware
  • • ModelCallLimitMiddleware
  • • RateLimitMiddleware
性能影响 1-5ms
