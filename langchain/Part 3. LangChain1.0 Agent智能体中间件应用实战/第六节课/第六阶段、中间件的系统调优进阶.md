## <center>第六阶段、中间件的系统调优进阶

### 1. 中间件编排黄金法则

#### 1.1 洋葱模型

```
        ┌────────────────────────────────────────┐
        │           Agent 执行核心               │
        │  ┌──────────────────────────────────┐  │
        │  │       wrap_tool_call            │  │
        │  │    ┌────────────────────────┐   │  │
        │  │    │   wrap_model_call     │   │  │
        │  │    │  ┌─────────────────┐  │   │  │
        │  │    │  │ before_model   │  │   │  │
        │  │    │  │  ┌─────────┐   │  │   │  │
        │  │    │  │  │before_  │   │  │   │  │
        │  │    │  │  │agent    │   │  │   │  │
        │  │    │  │  └─────────┘   │  │   │  │
        │  │    │  └─────────────────┘  │   │  │
        │  │    └────────────────────────┘   │  │
        │  └────────────────────────────────┘  │
        └────────────────────────────────────────┘

分层职责：
  外层 → 安全防护（Security, RBAC）
  中层 → 业务逻辑（Routing, Transform）
  内层 → 核心功能（Model Call, Tool Call）
  中心 → 基础服务（LLM, Tools）
```

#### 1.2 四大黄金法则

**法则一：安全类永远优先**

| 优先级 | 说明 |
|--------|------|
| 安全 > 业务 > 成本 > 监控 | 安全检查最先执行 |

**法则二：写操作按依赖排序**

如果 B 依赖 A 的输出结果，则 A 先执行：
- RBACMiddleware（注入用户信息）→ dynamic_prompt（使用用户角色）

**法则三：中断操作置后**

中断类中间件放在同类型最后执行：
```
before_agent 顺序：
[Security, RBAC, ..., Interrupt?]
                      ↑
                 中断操作放最后
```

**法则四：性能开销降序排列**

轻量级 → 重量级（快速失败）：

| 类型 | 性能开销 |
|------|----------|
| Monitor | <1ms |
| Enforce | 1-5ms |
| Control | 5-20ms |
| Modify | 1-50ms |

#### 1.3 执行顺序设计表

| 钩子类型 | 执行顺序 | 控制粒度 | 推荐实践 |
|----------|----------|----------|----------|
| before_agent | 正序 | 调用级 | 安全检查、初始化 |
| before_model | 正序 | 单次调用 | 输入验证、预算控制 |
| wrap_model_call | 嵌套 | 调用全程 | 路由、重试、缓存 |
| after_model | 逆序 | 单次调用 | 验证、日志 |
| wrap_tool_call | 嵌套 | 工具执行 | 参数校验、重试保护 |
| after_agent | 逆序 | 全局 | 清理、持久化 |

**注意**：
- before_* 和 wrap_* 正序执行
- after_* 逆序执行（最后注册的先执行）
- wrap_* 嵌套执行（形成包裹结构）

### 2. 安全优先原则实施

#### 2.1 分层安全检查体系

```
┌─────────────────────────────────────────────────────────┐
│                     分层安全检查                         │
├─────────────────────────────────────────────────────────┤
│  第1层：身份认证（before_agent 最外层）                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │  检查用户是否登录、Token 是否有效                │   │
│  │  → 未认证用户直接拒绝                           │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓                               │
│  第2层：输入验证（before_agent）                        │
│  ┌─────────────────────────────────────────────────┐   │
│  │  检查输入内容是否包含恶意代码、危险关键词        │   │
│  │  → 危险输入直接拒绝                             │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓                               │
│  第3层：业务授权（before_agent / before_model）         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  检查用户是否有权执行操作（RBAC）               │   │
│  │  → 无权限操作直接拒绝                           │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓                               │
│  第4层：合规检查（wrap_model_call）                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  检查操作是否符合法规要求（PII、敏感词）         │   │
│  │  → 不合规操作标记或拒绝                         │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

#### 2.2 各层实现示例

**第1层：身份认证**
```python
class AuthenticationMiddleware(AgentMiddleware):
    def before_agent(self, state, runtime):
        token = runtime.context.get("token")
        if not validate_token(token):
            return {"jump_to": "end", "error": "未认证"}
        return None
```

**第2层：输入验证**
```python
class InputValidationMiddleware(AgentMiddleware):
    def before_agent(self, state, runtime):
        dangerous_keywords = ["删除数据库", "hack", "攻击"]
        last_msg = state["messages"][-1].content
        for kw in dangerous_keywords:
            if kw in last_msg:
                return {"jump_to": "end", "error": "危险操作"}
```

**第3层：业务授权**
```python
class RBACMiddleware(AgentMiddleware):
    def before_agent(self, state, runtime):
        user_role = runtime.context.get("role")
        if not has_permission(user_role, operation):
            return {"jump_to": "end", "error": "权限不足"}
```

**第4层：合规检查**
```python
class ComplianceMiddleware(AgentMiddleware):
    def before_model(self, state, runtime):
        if contains_pii(state["messages"]):
            return {"messages": sanitize(messages)}
```

#### 2.3 冲突解决机制

**资源冲突**：多个中间件同时访问资源

| 方案 | 优点 | 缺点 |
|------|------|------|
| 锁机制 | 保证资源独占访问 | 降低并发性能 |
| 队列管理 | 提供公平的资源分配 | 增加延迟 |

**结果冲突**：多个中间件产生不同结果

| 方案 | 说明 |
|------|------|
| 权重决策 | 根据预定义规则选择结果 |
| 投票机制 | 多个中间件投票决定 |
| 最终决策者 | 指定某个中间件为最终决策者 |

#### 2.4 依赖关系管理

**依赖类型分析**：

1. **顺序依赖**：A → B（A 必须在 B 之前执行）
   - 例：身份认证 → 权限验证

2. **条件依赖**：A? → B 或 C（根据条件决定后续）
   - 例：VIP用户 → 高级处理，普通用户 → 标准处理

3. **数据依赖**：A 产生数据 → B 使用数据
   - 例：RBACMiddleware 注入权限 → dynamic_prompt 使用权限

**依赖排序策略**：

```
无依赖 → 首先执行
  ↓
单向依赖 → 按依赖关系排序
  ↓
条件依赖 → 特殊处理逻辑
  ↓
聚合依赖 → 等待所有相关完成
```
