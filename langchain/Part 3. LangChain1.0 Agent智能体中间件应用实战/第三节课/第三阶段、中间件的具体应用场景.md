## <center>第三阶段、中间件的具体应用场景

### 1. 中间件四大分类

中间件按照功能特性划分为四大类别，每种都有独特的定位和应用场景。

| **分类** | **核心功能** | **解决的问题** | **典型应用场景** |
|----------|-------------|----------------|-----------------|
| **Monitor (监控类)** | 观察执行状态、日志记录 | 调试困难、缺乏可观测性 | LangSmith tracing、性能监控、成本核算 |
| **Modify (修改类)** | 修改输入/输出、上下文管理 | 上下文窗口溢出、Prompt优化 | SummarizationMiddleware、ContextEditingMiddleware |
| **Control (控制类)** | 流程阻断、人工介入 | AI幻觉、高风险操作失控 | HumanInTheLoopMiddleware、ToolRetryMiddleware |
| **Enforce (强制类)** | 安全过滤、限流、合规检查 | 数据泄露、API滥用 | PIIMiddleware、ModelCallLimitMiddleware |

![中间件四大分类](../assets/第二课_01_中间件四大分类.html)

**四大分类详解**：

- **Monitor（监控类）**：系统的"观察者"，默默记录系统运行的每一个细节。性能监控不仅记录响应时间和资源使用，还能自动检测性能异常。成本追踪精确记录每个 Token 的消耗。

- **Modify（修改类）**：系统的"优化师"，主动改变数据处理方式。智能摘要中间件通过复杂算法分析对话内容，保留关键信息，过滤冗余内容。

- **Control（控制类）**：系统的"指挥官"，管理系统的行为，确保操作在预定规则范围内。人工介入中间件识别敏感操作，保持系统自动化的同时确保安全性。

- **Enforce（强制类）**：系统的"守护者"，保护系统免受威胁。安全强制执行中间件对输入输出进行全方位安全检查。

### 2. 深入生命周期 - 中间件的 6 个切入点 (Hooks)

#### 2.1 Hook 执行顺序的深层逻辑

中间件的 6 个 Hook 点构成完整的生命周期：

| **Hook 点** | **作用** | **执行阶段** |
|-------------|---------|-------------|
| before_agent | Agent 执行前最后机会修改输入 | 最先执行 |
| before_model | 模型调用前修改 prompt / 选择模型 | 次之 |
| wrap_model_call | 包裹模型调用（异常处理、重试） | 嵌套执行 |
| wrap_tool_call | 包裹工具调用（异常处理、重试） | 嵌套执行 |
| after_model | 模型调用后修改输出 | 逆序执行 |
| after_agent | Agent 执行完成后的最终处理 | 最后执行 |

![中间件洋葱模型](../assets/第一课_01_中间件洋葱模型.html)

#### 2.2 数据传递机制

Hook 间通过两种机制传递数据：

1. **AgentState（全局状态容器）**：贯穿整个 Agent 执行流程，跨多次模型调用持久化
2. **ModelRequest/Response（单次调用）**：仅作用于单次模型/工具调用，可直接修改

### 3. 案例：中间件配置示例

```python
from langchain.agents import create_agent
from langchain.agents.middleware import (
    SummarizationMiddleware,
    HumanInTheLoopMiddleware,
    ToolRetryMiddleware
)

# 配置多中间件
agent = create_agent(
    model=model,
    tools=[search_tool, calculator],
    middlewares=[
        SummarizationMiddleware(),      # Modify - 上下文压缩
        ToolRetryMiddleware(max_retries=3),  # Control - 重试机制
        HumanInTheLoopMiddleware(),   # Control - 人工审批
    ]
)
```

**配置要点**：
- `before_*` 正序执行（1 → 2 → 3）
- `wrap_*` 嵌套执行
- `after_*` 逆序执行（3 → 2 → 1）
