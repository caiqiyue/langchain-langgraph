批处理 vs 流式传输 vs 异步并发

📦
batch()
Batch Processing
同步批量调用，一次性提交多个请求，模型并行处理后返回所有结果
# 批量提交，同时处理 responses = model.batch([ "问题1", "问题2", "问题3" ])
🌊
stream()
Streaming Output
流式返回，逐块获取结果，像打字效果一样实时显示内容
# 逐块获取，实时显示 for chunk in model.stream("长文本"): print(chunk.content, end="")
⚡
abatch()
Async Batch
异步批量并发处理，配合 RunnableConfig 控制并发数
config = RunnableConfig(max_concurrency=3) responses = await model.abatch( [...], config=config )

📊 特性对比表

特性 batch() stream() abatch()
执行方式 同步 流式 异步
返回时机 全部完成返回 逐块返回 全部完成返回
并发控制 无 不适用 max_concurrency
适用场景 文档摘要、批量问答 聊天机器人、实时显示 高并发生产环境
