Message 消息类型详解

⚙️
SystemMessage — 系统消息
设定模型的身份、风格、规则，是"最高优先级"的指令
SystemMessage(content="你是专家")
👨‍💻
Developer Message
开发者提示，模型的功能逻辑 / 工程约束（OpenAI 新增 role）
{"role": "developer", ...}
👤
HumanMessage — 用户消息
表示用户提问内容，是本轮对话的主体输入
HumanMessage(content="问题")
🤖
AIMessage — 助手消息
表示模型历史回答，有助于形成上下文记忆
AIMessage(content="回复")
🔧
ToolMessage — 工具消息
工具调用结果（用于 Agent 场景），传递工具执行后的返回值
ToolMessage(content="结果")

📌 消息执行顺序与优先级（非常重要）

LLM 按以下顺序解析消息，模型永远会参考全部 messages 才得出最终输出

1
system
最高优先级
2
developer
功能逻辑
3
user
当前输入
4
assistant
历史对话
5
tool
工具调用
