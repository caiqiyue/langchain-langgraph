结构化输出解析器对比

让大模型返回可程序解析的数据

📝
StrOutputParser
基础解析
将模型输出解析成纯字符串（默认解析器）
chain = prompt | model | StrOutputParser() result = chain.invoke("问题") # result 是字符串
{ }
JsonOutputParser
JSON 结构化解析
将 LLM 输出强制解析为 JSON 格式
parser = JsonOutputParser(pydantic_object=WeatherInfo) chain = prompt | model | parser result = chain.invoke(...) # result 是 dict 对象
🐍
with_structured_output
Pydantic 结构化输出
使用 Pydantic 模型定义严格数据结构
class Person(BaseModel): name: str age: int structured_llm = llm.with_structured_output(Person) result = structured_llm.invoke(prompt) # result 是 Person 对象

🔑 结构化输出关键要点

  • 1 提示词必须包含 "json" 关键词 — DeepSeek API 要求提示词中包含 "json" 这个词，否则会报错
  • 2 推荐方案对比 — JsonOutputParser 最简洁；with_structured_output 功能最强大；手动 JSON 解析最稳定
  • 3 配置建议 — 设置 temperature=0.0 获得更稳定的输出；提供清晰的 JSON 格式示例
  • 4 常见错误 — 提示词中没有 "json" 关键词；没有设置低温度参数；没有提供 JSON 格式示例
