## <center>第五阶段、工具（Tools）的集成与调用</center>

&emsp;&emsp;工具是Agent与外部世界交互的桥梁。在LangChain中，工具的`name`、`description`和`args_schema`至关重要，它们共同决定了模型是否以及如何选择和调用工具。一个设计良好的工具描述是提示工程的关键部分。

### 1. 工具定义三种方式

![工具定义三种方式对比](../assets/第五课_01_工具定义三种方式.html)

**图解说明**：三种工具定义方式对比 — @tool装饰器最简洁，StructuredTool.from_function()支持强类型校验，继承StructuredTool完全自定义

#### 1.1 @tool 装饰器（最简单）

```python
from langchain_core.tools import tool

@tool
def multiply(a: int, b: int) -> int:
    """计算两个整数的乘积"""
    return a * b
```

**特点**：

- 代码极简，一行装饰器
- 自动推断参数类型
- 适合快速原型验证

#### 1.2 StructuredTool.from_function()（生产级）

```python
from pydantic import BaseModel, Field
from langchain_core.tools import StructuredTool

class DivideInput(BaseModel):
    dividend: float = Field(description="被除数")
    divisor: float = Field(description="除数，不能为零")

division_tool = StructuredTool.from_function(
    func=divide,
    name="DivisionTool",
    description="安全执行除法运算",
    args_schema=DivideInput,
)
```

**特点**：

- Pydantic 参数校验
- 完整元数据配置
- 支持异步执行

#### 1.3 继承 StructuredTool（完全自定义）

```python
from langchain_core.tools import StructuredTool

class OrderQueryTool(StructuredTool):
    name: str = "query_order"
    description: str = "查询电商平台订单"

    def _run(self, order_id: str) -> dict:
        # 自定义业务逻辑
        return result
```

**特点**：

- 完全自定义行为
- 支持状态管理
- 适合复杂业务逻辑

### 2. 工具选择策略

当 Agent 面对多工具场景时，需要智能路由机制：

#### 工具分组策略

```python
TOOL_GROUPS = {
    "search": [search_news, search_wiki],
    "calc": [add, subtract, multiply],
}

# 意图识别后，按需加载工具组
```

#### 工具描述的重要性

- 工具名称必须**动词开头**
- 每个工具使用标准化 schema
- 工具描述必须包含：能干什么、不能干什么、典型输入示例
