⚙️ create_agent 九大参数

🔥 必填参数
必填 model
str / LLM 实例
推理引擎，决定 Agent 的"智力水平"。
生产环境建议使用实例化配置。
必填 tools
list[BaseTool]
工具列表，决定 Agent 的"能力范围"。
工具描述（description）是 LLM 选择工具的依据。
🌿 可选参数

system_prompt

str

Agent 的"人格说明书"，定义角色、行为准则、输出格式

middleware

list[Middleware]

功能扩展：日志记录、安全检查、人工介入、上下文压缩

checkpointer

MemorySaver

短期记忆，管理会话级对话历史（与 thread_id 绑定）

store

BaseStore

长期记忆，跨会话持久化用户信息、偏好等

state_schema

TypedDict

自定义状态结构，扩展 Agent 的状态字段

context_schema

TypedDict

动态上下文结构，运行时传入用户角色、权限等

response_format

BaseModel

结构化输出格式，适用于 API 返回标准化场景

💡 最佳实践

system_prompt 应简洁明了，定义清楚 Agent 的角色和约束。
checkpointer 生产环境必须配置，否则每次都是新对话。
tools 的描述（description）是 LLM 选择工具的唯一依据，必须清晰准确。
