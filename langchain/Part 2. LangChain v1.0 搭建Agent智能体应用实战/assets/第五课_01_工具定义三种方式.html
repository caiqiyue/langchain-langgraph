🔧 工具定义三种方式对比

🎯
@tool 装饰器
最简单，一行搞定

✅ 优点

  • 代码极简，一行装饰器
  • 自动推断参数类型
  • 快速原型验证

❌ 局限

  • 参数控制较弱
  • 不支持异步
  • 不适合复杂参数
代码简洁度
⭐⭐⭐⭐⭐
@tool def multiply(a: int, b: int) -> int: """计算两数乘积""" return a * b
🏗️
StructuredTool
from_function()

✅ 优点

  • Pydantic 参数校验
  • 完整元数据配置
  • 支持异步执行

❌ 局限

  • 代码量中等
  • 需要 Pydantic 模型
生产适用度
⭐⭐⭐⭐⭐
from pydantic import BaseModel class Input(BaseModel): a: int b: int StructuredTool.from_function( func=multiply, name="multiply", args_schema=Input )
🔬
继承 StructuredTool
完全自定义

✅ 优点

  • 完全自定义行为
  • 支持状态管理
  • 复杂业务逻辑

❌ 局限

  • 代码最复杂
  • 需要更多维护
灵活度
⭐⭐⭐⭐⭐
class MyTool(StructuredTool): name: "my_tool" def _run(self, param): return result
