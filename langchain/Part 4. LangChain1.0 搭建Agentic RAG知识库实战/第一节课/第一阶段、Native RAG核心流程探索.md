## <center>第一阶段、Native RAG核心流程探索</center>

### 1. RAG 基础概念

RAG = Retrieval（检索）+ Augmented（增强）+ Generation（生成）

核心思想：让 LLM 在回答问题前，先从知识库中检索相关信息，将检索结果作为上下文加入提示词，从而生成更准确的答案。

![RAG基础流程](../assets/第一课_01_RAG基础流程.html)

**图解说明**：RAG 两阶段 — 阶段一（知识库构建离线）→ 阶段二（问答在线）

### 2. RAG vs 普通 LLM

| 维度 | 普通 LLM | RAG |
|------|----------|-----|
| 知识来源 | 仅训练数据 | 训练数据 + 知识库 |
| 实时信息 | 无法获取 | 可检索 |
| 幻觉问题 | 较严重 | 通过检索减少 |
| 知识更新 | 需重新训练 | 更新知识库即可 |

### 3. 两阶段过程

**阶段一：知识库构建（离线）**

```
文档 → 解析 → 切分 → 向量化 → 存储到向量数据库
```

**阶段二：问答（在线）**

```
用户提问 → 问题向量化 → 相似度检索 → 构建增强提示 → LLM生成
```
