# 学习资料优化处理过程详细记录

## 项目概述

将原始 Jupyter Notebook 学习资料转化为结构化、图文并茂的自学文档。

**原始资料来源**：
- `Part X. XXX/*.ipynb` (Jupyter Notebook 格式)
- 包含大段落文本的课程内容

**目标格式**：
- `Part X. XXX/第N节课/XX_名称.py` (Python 代码案例)
- `Part X. XXX/第N节课/第N阶段、内容名.md` (结构化 MD 文档)
- `Part X. XXX/assets/*.html` (HTML 可视化图表)

---

## 第一部分：原始资料分析方法

### 1.1 分析 Jupyter Notebook 内容结构

原始 Notebook 包含两种主要形态：
1. **代码单元**：Python 代码示例
2. **Markdown 单元**：大段落教学内容

### 1.2 识别知识分割点

从 MD 文档中识别：
- 标题层级（# ## ###）
- 自然段落分隔
- 代码示例与文字说明的边界
- 表格和列表的聚团

---

## 第二部分：处理步骤详解

### Step 1: 创建 Python 代码案例

#### 文件命名规则
```
Part X. XXX/第N节课/XX_名称.py
```

#### Python 代码模板

```python
# -*- coding: utf-8 -*-
"""
【案例 N】案例标题
============================================

本案例展示...

要点：
1. 要点一
2. 要点二
"""

# ============================================================
# 1. 环境配置
# ============================================================
import os
from dotenv import load_dotenv

load_dotenv(override=True)

# ... 配置代码

# ============================================================
# 2. 核心功能实现
# ============================================================
# ... 实现代码

# ============================================================
# 3. 执行与验证
# ============================================================
# ... 测试代码
```

#### 关键规范
- 顶部包含案例标题和要点说明
- 使用 `# ============================================================` 作为分区线
- 中文注释丰富
- 无硬编码 API Key（从 .env 读取）

### Step 2: 生成 HTML 可视化图表

#### 使用技能：axi-front-design

**技能路径**：`userSettings:axi-front-design`

#### 图表生成流程

1. **确定图表内容**
   - 分析 MD 文档中的流程/架构描述
   - 识别需要可视化的核心概念

2. **使用 AskUserQuestion 确认设计参数**
   - 语言：中文 / 英文 / 中英混排
   - 字号强度：规范级 / 震撼态 / 克制型
   - 风格方案数量：1-3 版

3. **设计 HTML 页面**
   - 遵循 skill 中的 CSS 规范
   - 使用 React + Babel 实现交互
   - 关键布局规范：
     - 不要用负 margin 堆叠中文
     - `::after` 作为 flex 子成员时注意数量
     - 适当留白

4. **保存位置**
   ```
   Part X. XXX/assets/图表名称.html
   ```

#### HTML 嵌入 MD 方式
```markdown
![图名](../assets/文件名.html)
```

### Step 3: 重组 MD 文档

#### MD 文档结构模板

```markdown
## <center>第N阶段、内容标题</center>

### 1. 小节标题

#### 1.1 子小节

**加粗说明**：内容描述

| 表格 | 说明 |
|------|------|
| 字段1 | 说明1 |

```python
# 代码示例
```

![可视化图表](../assets/图表名.html)

**图解说明**：图表内容说明

### 2. 拓展思考

#### 2.1 深度概念解析

&empsp;&empsp;扩展内容...

#### 2.2 工程实践

&empsp;&empsp;实践建议...
```

## 第四部分：关键技术细节

### 4.1 HTML 图表设计规范

#### 布局原则
- 1920×1080 标准尺寸
- 正文 ≥ 24px
- 标题 64–80px

#### CSS 反陷阱
```css
/* ✅ 正确：中文大标题用 grid 分列 */
.cover { display: grid; grid-template-columns: 280px 1fr; gap: 80px; }

/* ❌ 错误：负 margin 堆叠中文 */
.title { margin-top: -20px; }
```

#### Flex 伪元素规范
```css
/* ✅ 正确：规尺线作为独立 DOM 元素 */
.row { display: flex; align-items: center; gap: 20px; }
.row .rule { flex: 1; height: 1px; background: #000; }

/* ❌ 错误：::after 作为 flex child 会打乱布局 */
.row::after { content: ""; flex: 1; } /* 会变成第三个 flex item */
```

### 4.2 MD 文档编写规范

#### 表格格式
```markdown
| 字段 | 说明 |
|------|------|
| 值1 | 说明1 |
```

#### 代码块格式
```python

```

#### 图片嵌入格式
```markdown

```

### 4.3 拓展思考内容编写

每个阶段的拓展思考应包含：

## 第八部分：总结

### 处理原则
1. **结构化**：大段落拆分为小单元
2. **可视化**：关键概念配 HTML 图表
3. **深度化**：添加拓展思考内容
4. **可执行**：代码案例可直接运行

- Part 5: 5 个阶段 MD 文档（含代码和拓展思考）
- Part 6: 4 个阶段 MD 文档（含代码和拓展思考）

### 核心差异：原始 vs 优化后

| 维度 | 原始资料 | 优化后 |
|------|----------|--------|
| 格式 | Jupyter Notebook | MD + Python + HTML |
| 结构 | 大段落 | 小单元分层 |
| 图表 | 无 | HTML 可视化 |
| 代码 | 嵌入 Notebook | 独立 .py 文件 |
| 深度 | 基础内容 | 含拓展思考 |
| API Key | 可能硬编码 | 从 .env 读取 |
